## Top picks

### 1. [SLYP: Agentic Vulnerability Reasoning on Windows COM Binaries](https://arxiv.org/abs/2605.05000)
**Authors:** Hwiwon Lee, Jongseong Kim, Lingming Zhang (UIUC) · arXiv cs.CR · submitted 2026-05-06

SLYP is an end-to-end agentic pipeline that discovers race-condition vulnerabilities in Windows Component Object Model (COM) binaries and synthesizes debugger-verified proof-of-concept exploits. COM services run with elevated privileges and are widely reachable by authenticated users, making them a high-value surface for local privilege escalation. The system exposes three tool interfaces — binary exploration, COM activation inspection, and dynamic debugger feedback — that let the agent move from static context to confirmed, reproducible PoC in a single loop. On a benchmark of 20 COM objects (40 vulnerability cases) it achieves 0.973 F1, outperforming production coding agents by up to 0.208 F1 and the SOTA static analyzer by 3.3×. Deployed against production Windows services, SLYP discovered 28 previously unknown vulnerabilities across nine COM services, all confirmed by MSRC, earning 16 CVEs and $140,000 in bug bounties.

**Methodology:** Agentic pipeline with static binary analysis, COM inspection metadata, and dynamic debugger-feedback loop; PoC generation verified against a live debugger before reporting.

**Why it matters:** This is the most concrete demonstration this week that agentic PoC generation translates from benchmarks to real-world critical-path software — with MSRC confirmation and bounty numbers to back it up. Essential reading if you work on automated exploit generation or evaluation methodology.

### 2. [Root-Cause-Driven Automated Vulnerability Repair](https://arxiv.org/abs/2605.04251)
**Authors:** Hulin Wang, Zion Leonahenahe Basque, Jie Hu, Ati Priya Bajaj, Yibo Liu, Samuel Zhu, Giorgi Kobakhia, Nikhil Chapre, Will Rosenberg, Siddharth Mishra, Aditya Maheshbhai Gabani, Moritz Schloegel, Adam Doupé, Yan Shoshitaishvili, Ruoyu Wang, Tiffany Bao · arXiv cs.CR · submitted 2026-05-05

The system, **Kumushi**, addresses a persistent failure mode in LLM-based patch generation: without an explicit root-cause signal, agents drift toward shallow edits that silence observed crashes while leaving the underlying defect intact. Kumushi adds a root-cause localization step before patch synthesis and validates each candidate against replay-based test oracles that confirm both the crash is fixed and the original trigger no longer fires. Under tier-1 oracle evaluation, Kumushi reaches an 85% plausible-patch rate, outperforming PatchAgent (72%) and LoopRepair (2%), and produces significantly more root-cause fixes relative to symptom patches. The paper argues that the localization-before-synthesis discipline is the key differentiator, not model size or prompt engineering.

**Methodology:** LLM agent with explicit root-cause localization stage, patch candidate synthesis, and replay-oracle validation; evaluated on a public vulnerability repair benchmark under tier-1 oracle conditions.

**Why it matters:** If your research touches automated patching or APR evaluation, Kumushi's 85% result at tier-1 is the new number to beat, and the root-cause framing gives a clear diagnostic lens for why prior systems underperform.

### 3. [When Prompts Become Shells: RCE Vulnerabilities in AI Agent Frameworks](https://www.microsoft.com/en-us/security/blog/2026/05/07/prompts-become-shells-rce-vulnerabilities-ai-agent-frameworks/)
**Authors:** Uri Oren, Amit Eliahu, Dor Edry — Microsoft Defender Security Research Team · Microsoft Security Blog · 2026-05-07

Microsoft's security research team discloses two critical vulnerabilities in Semantic Kernel, its open-source AI agent framework (~27K GitHub stars). **CVE-2026-26030** exploits unsafe string interpolation in the In-Memory Vector Store's default filter: the input is LLM-controlled and unvalidated, so an attacker can close quote characters and inject Python that traverses the class hierarchy to `BuiltinImporter` and executes arbitrary system commands — a classical `eval()`-as-a-gate failure. **CVE-2026-25592** exploits a `[KernelFunction]` attribute accidentally left on `DownloadFileAsync`, making the function AI-callable and exposing `localFilePath` to attacker control; the demonstrated chain writes a payload to the Windows Startup folder for persistence. The fixes are a four-layer AST-allowlist approach (replacing the blocklist) plus path canonicalization. The core lesson: "Your LLM is not a security boundary. The tools you expose define your attacker's affected scope."

**Methodology:** Manual source-code review of Semantic Kernel; PoC development demonstrating full RCE chain; four-layer defensive fix deployed in versions 1.39.4 / .NET SDK 1.71.0.

**Why it matters:** A concrete case study of how agent framework architecture choices — not model failures — introduce execution risk; directly useful as a reference when you design or audit tool-use interfaces in your own agent pipelines.

### 4. [Patch2Vuln: Agentic Reconstruction of Vulnerabilities from Linux Distribution Binary Patches](https://arxiv.org/abs/2605.06601)
**Authors:** Isaac David, Arthur Gervais · arXiv cs.CR · submitted 2026-05-07

Patch2Vuln asks whether an LLM agent, restricted entirely to local binary-derived evidence, can reconstruct the security meaning of Linux distribution updates — useful for defenders who receive package updates before CVE details are published. The pipeline extracts old/new ELF binary pairs from `.deb` deltas, diffs them with Ghidra and Ghidriff, ranks changed functions by heuristic security relevance, builds candidate dossiers per function, and passes them to an offline LLM agent that produces a preliminary audit, a bounded validation plan, and a final security verdict. On a 25-case Ubuntu benchmark (20 security-update pairs, 5 controls), the agent localizes 10 of 20 security targets and correctly rejects all 5 controls. The authors identify binary-diff coverage and the lack of dynamic behavioral validation as the primary bottlenecks.

**Methodology:** Local, resumable pipeline: Ghidra/Ghidriff binary diffing → function ranking → offline LLM agent for vulnerability audit, without network access or CVE metadata.

**Why it matters:** Opens a realistic, under-explored scenario — pre-CVE binary patch triage — with a clean reproducible benchmark; the offline-agent constraint is also a useful design discipline for air-gapped or privacy-sensitive settings.

### 5. [The Anthropic Institute Research Agenda](https://www.anthropic.com/research/anthropic-institute-agenda)
**Source:** Anthropic · 2026-05-07

Anthropic published the research agenda for The Anthropic Institute (TAI), an internal organization that uses frontier-lab access to study AI's societal impact. The four pillars are: Economic diffusion, **Threats and resilience** (covering cyber, bio, and surveillance), AI systems in the wild (agents, governance, reliability, human-AI interaction), and AI-driven R&D. The threats pillar explicitly targets AI-enabled automated hacking and building early warning systems. This announcement comes two weeks after the Claude Mythos Preview disclosed that a frontier model can autonomously find thousands of zero-days in every major OS and browser — a capability Anthropic is not releasing generally and has gated through Project Glasswing (~40 critical-software organizations).

**Methodology:** Institutional research agenda publication; programmatic research in four areas using internal frontier-model access.

**Why it matters:** TAI is the first lab-internal institute explicitly tasked with studying AI-enabled cyber threats at scale; its output is likely to shape both benchmark design and responsible-disclosure norms for AI-assisted vulnerability research.

## Also notable

### [AgenticVM: Agentic AI for Adaptive Software Vulnerability Management](https://arxiv.org/abs/2605.01739)
**Authors:** Researchers from Adelaide University and CSIRO Data61 · arXiv cs.CR · submitted 2026-05-03.
A modular multi-agent framework that integrates LLMs with conventional security tooling (NVD, EU-VDB) to automate the full vulnerability management lifecycle — detection, assessment, prioritization, and remediation. A BERT-based CVSS prediction module fills in missing severity attributes (89.3% accuracy), and LLM-driven agents then collapse raw scanner output into actionable queues, achieving up to 98% alert reduction (3,983 findings → 82 high-priority items). The paper also covers architectural trade-offs for real-world deployment. Solid engineering contribution; not a frontier research contribution, but a practical reference for integrating LLMs into existing VM workflows.

### [HackerSignal: A Large-Scale Multi-Source Dataset Linking Hacker Community Discourse to the CVE Vulnerability Lifecycle](https://arxiv.org/abs/2605.03158)
**arXiv cs.CR · submitted ~2026-05-04.**
HackerSignal aggregates 7.45 million deduplicated documents from 64 public sources across eight layers — hacker forums, exploit archives, vulnerability advisories, bug-bounty disclosures, fix commits, exploitation references — spanning 36 years (1990–2026). It maps the full exploit-to-fix trajectory via a shared CVE identifier space. Three benchmark tasks are defined: CVE linkage retrieval, 8-class exploit-type classification with temporal OOD evaluation, and prospective generalization where train/test CVE sets are disjoint. Addresses a real gap in CTI datasets — most existing corpora cover only advisories or commits, not the hacker-community discourse that often signals exploitation intent early. Useful as a pre-training or evaluation corpus for any model that reasons over CTI or exploit timelines.

### [MAGE: Safeguarding LLM Agents against Long-Horizon Threats via Shadow Memory](https://arxiv.org/abs/2605.03228)
**Yuhui Wang, Tanqiu Jiang, Jiacheng Liang, Charles Fleming, Ting Wang (Stony Brook / Cisco) · arXiv cs.CR · submitted 2026-05-04.** *(Also in previous digest, 2026-05-07.)*
Inspired by the shadow stack abstraction in systems security, MAGE maintains a dedicated safety-focused memory alongside the agent's working memory. This shadow memory distills safety-critical context across the full execution trajectory and is consulted before each pending action. The key threat model is "long-horizon" attacks that spread their malicious setup across many benign-looking turns so that no single turn triggers a safety check. Evaluation shows MAGE substantially outperforms existing single-turn defenses on detection accuracy across diverse long-horizon attack scenarios.

**Graham Sutherland, Paweł Płatek · Trail of Bits Blog · 2026-05-05.**
Detailed solutions to two security challenge programs from the Trail of Bits Testing Handbook. The Linux case dissects a ping program with an `inet_ntoa` global-buffer gotcha and a command injection vulnerability. ### [ProgramBench: Evaluating LM Software Reconstruction](https://www.emergentmind.com/papers/2605.03546)
**arXiv cs.SE · submitted 2026-05-05 *(date uncertain)*.**
Introduces a 200-task benchmark for measuring whether software engineering agents can holistically reconstruct software (ffmpeg, ripgrep, jq, SQLite, interpreters, codecs, CLI tools). Evaluation uses agent-driven behavioral fuzzing to generate end-to-end tests without prescribing implementation structure. Primarily an SE-agent benchmark rather than a security one, but its fuzzing-based evaluation methodology is directly transferable to security-focused agent evaluations where oracle-free correctness is needed.
